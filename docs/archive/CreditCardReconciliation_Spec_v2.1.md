# CORPORATE CARD EXPENSE RECONCILIATION SYSTEM
## Technical Specification v1.0 — Google Apps Script
**Date:** April 2026 | **Platform:** Google Apps Script (V8) | **Status:** RELEASE v1.0

### Changelog v1.0 (Current)
- **Folder Refactoring:** Switched to a single-level `DONE-MM-YYYY` folder structure for matched receipts.
- **Root-First Unmatching:** Unmatched receipts now remain in the root Drive folder for easier manual sorting.
- **Smart Auto-Fill:** System now automatically populates Account Code (AI-suggested), I/O Code, Cost Center, and Tax ID (from Config defaults) upon successful match.
- **Pure DD-MM Containers:** DD-MM sheets are now created completely blank, serving as clean containers for users to paste images into.
- **Full Scanned Audit:** Audit log now captures 100% of scanned files, marking those missed by AI as "AI Skip".

### Changelog v0.9 (Pre-release)
- **Enhanced Matching Logic:** Implemented aggressive character stripping for merchant names and refined scoring weights.
- **Audit Reasons:** Added a "Match Reason" column to the `OCR Audit` sheet (without Raw OCR text for cleaner logs).
- **Numeric Index Mapping:** Switched from Drive IDs to numeric index mapping for robust AI response parsing.

## 1C. Sheet Structure — Complete Workbook

### SHEET: `Corporate card details` (Primary sheet — always exists)

```
Row 1:  Header row (24 columns A–X, fixed)
Row 2+: Transaction data

ZONES:
  Col A–P  (16 cols): BANK DATA — imported from bank file, protected from editing after import
  Col Q–V  (6 cols):  USER DATA — system pre-fills defaults, user confirms/overrides

CONSTRAINTS:
  - Col A–P: warn-only protection after import (user can still edit if needed)
  - Col Q:   Data Validation dropdown from Config Account Codes table
  - Col R:   VLOOKUP formula =IFERROR(VLOOKUP(Q2, Config!$AccountCodeRange, 2, 0), "")
  - Col S, T, V: pre-filled with Config defaults on import, user may override
  - Col W:   only accepts STATUS.* values (enforced by system, no manual dropdown)
  - Col X:   system-written string reference, read-only
  
FORMATTING:
  - Col W conditional formatting:
      MATCHED    → green fill  (#E6F4EA)
      NO_INVOICE → red fill    (#FCE8E6)
      AMBIGUOUS  → yellow fill (#FEF9C3)
      PENDING    → no fill (default)
```

## 2C. Error Handling Matrix

| Situation | Detection | System Action | User Action Required |
|---|---|---|---|
| Missing Config (B1/B2/B3 blank) | `getConfig()` on run | Alert describing which field is missing, halt | Fill Config |
| Invalid Service Account JSON | JSON.parse failure | Alert "Invalid JSON: ..." | Re-paste Service Account JSON |
| Cloud Vision HTTP 429 (quota) | Response code | Retry once after 2s; if still fails → move image to /Error/ | Wait 1 min, re-run |
| Image too blurry / no text | fullText empty after OCR | Move image → /Error/, log "no text detected" | Re-photograph |
| Gemini returns invalid JSON | `parseGeminiBatchResponse()` | Log raw response, skip this image, move → /Error/ | Check logs, re-run |
| No statement rows match invoice | `matchToStatement()` | Move image → /Unmatched/, log reason | Manually fill Col U/W |
| ≥2 statement rows match same invoice | `matchToStatement()` | Col W = AMBIGUOUS, sidebar lists candidates | Pick correct row |
| Date diff > 2 days | `matchToStatement()` score | No match, image → /Unmatched/ | Manually fill if correct |
| Currency mismatch | `matchToStatement()` check | No match (VND ≠ KRW even if same number) | Manually fill if needed |
| User re-imports: DD-MM sheet exists | `createOrUpdateDaySheet()` | Update summary table only, NEVER delete images | Nothing |
| Col Q blank at export | `validateBeforeExport()` | Warning: "N rows missing Account Code" | Fill before copying |
| GAS 6-minute timeout | Execution limit | Log "Processed X/Y images — timeout. Re-run to continue." | Re-run menu action |
| Images inserted "In cell" not found | `extractEmbeddedImages()` | Log "0 images found. Use Insert → Image → Over cells." | Re-insert images correctly |
| Drive Folder inaccessible | `getOrCreateMonthFolder()` | Alert with folder ID, halt | Check DRIVE_FOLDER_ID and permissions |

# DOCUMENT 3 — CLAUDE-READY CODING PROMPT

```
================================================================================
CODING PROMPT — Corporate Card Expense Reconciliation System v2.1
Platform: Google Apps Script (V8 runtime)
Language: ALL English — code, comments, UI labels, alerts, log messages, menu items
Based on: actual bank export file (Korean corporate card — 16 columns A–P)

## SYSTEM OVERVIEW
Google Apps Script integrated into Google Sheets that:
1. Imports corporate card bank statements (Excel, 16 cols A–P)
2. Auto-creates per-day sheets "DD-MM" with transaction summary
3. Users manually insert invoice photos into DD-MM sheets (Insert → Image → Over cells)
4. System reads all embedded images from a sheet → OCR each via Cloud Vision (per image)
   → aggregates ALL OCR texts → sends ONE Gemini call per sheet (batch architecture)
   → parses all invoices at once → matches to statement rows (deterministic GAS logic)
5. Fills Col U (Summary), Col W (Match Status), Col X (Invoice Reference) automatically
6. Account Code (Col Q) links to Account Name (Col R) via Config VLOOKUP
7. I/O, Cost Center, Tax ID from Config defaults, user-overridable
8. Output: 24-column table user copies to company Excel report

## CRITICAL ARCHITECTURE RULE — BATCH AI PATTERN
  Cloud Vision: called ONCE PER IMAGE (fast OCR, cheap, parallel-friendly)
  Gemini:       called ONCE PER SHEET (receives ALL OCR texts aggregated → returns ALL parsed results)
  Matching:     pure deterministic GAS code (zero AI involvement)

  WRONG pattern (DO NOT DO):
    for each image:
      ocrText = callCloudVision(image)      ← ok
      parsed  = callGemini(ocrText)         ← WRONG: N Gemini calls for N images

  CORRECT pattern:
    ocrResults = []
    for each image:
      ocrResults.push(callCloudVision(image))   ← N Vision calls (cheap)
    batchResult = callGeminiBatch(ocrResults)    ← 1 Gemini call for all images

## LANGUAGE REQUIREMENT
  ALL strings in the system must be English:
  - Menu item labels
  - Alert/dialog messages
  - Log messages
  - Sheet headers
  - Status values (STATUS.* constants)
  - Error messages returned to user
  - Code comments and variable names
  - Config sheet table headers
  NO Vietnamese, NO Korean anywhere in the codebase.

### Group: MATCHING ENGINE (pure deterministic — zero AI)
- `phaseD_applyResults(mainSheet, geminiResults, statementRows, sheetDate, driveRootFolder, sheetName)` → summary
  - For EACH geminiResult:
    - Call `matchToStatement(result, statementRows)`
    - Branch on result type:
        match found    → `applyMatch(mainSheet, match.row, sheetName, result.imageIndex, result)`
                          → move image to Drive day folder
        no match       → log "No match | ..." → move to /Unmatched/
        ambiguous      → `applyAmbiguous(mainSheet, candidates, sheetName, result.imageIndex)`
        error          → log and skip
  - Return {matched, unmatched, ambiguous, errors} counts

- `matchToStatement(parsedInvoice, statementRows)` → {row, score} | {ambiguous, candidates} | null
  - Input: parsedInvoice = {date, amount, currency, ...}
  - Input: statementRows = array of {rowNum, date, amount, currency, status}
  - Filter: exclude rows where status !== STATUS.PENDING
  - Score each row:
      currency match (case-insensitive exact):  REQUIRED — score = 0 if mismatch
      amount === parsedInvoice.amount:           +0.6 (exact, no tolerance)
      dateDiff === 0:                            +0.4
      dateDiff === 1:                            +0.3
      dateDiff === 2 (= MATCH_DATE_TOLERANCE):  +0.2
      dateDiff > MATCH_DATE_TOLERANCE:           0 (disqualify)
  - Accept threshold: score >= MIN_MATCH_SCORE (0.8)
  - 0 qualified → return null
  - 1 qualified → return {row: rowNum, score}
  - ≥2 qualified → return {ambiguous: true, candidates: [rowNums]}

- `applyMatch(sheet, rowNum, daySheetName, imageIndex, invoiceData)` → void
  - Build 1×3 update: [[invoiceData.summary, STATUS.MATCHED, `${daySheetName} #${imageIndex}`]]
  - `sheet.getRange(rowNum, COL.SUMMARY, 1, 3).setValues(update)`

- `applyAmbiguous(sheet, candidateRows, daySheetName, imageIndex)` → void
  - For each candidate row: set Col W = STATUS.AMBIGUOUS
  - Log candidates to sidebar for user review

## PLATFORM CONSTRAINTS & NOTES
- GAS max execution time: 6 minutes
  - Phase A (Cloud Vision per image): ~2s per image × 50 images = ~100s — within limit
  - Phase C (1 Gemini batch call): ~5–15s — within limit
  - Phase D (matching + Drive moves): ~1s per image — within limit
  - If approaching timeout: save progress in PropertiesService, alert user to re-run

- XLSX parsing in GAS:
  - GAS cannot natively parse .xlsx binary
  - Approach: use HtmlService to read file in browser via FileReader API → send base64 to server
  - Server decodes base64 → writes temp file → creates temp Spreadsheet via Sheets API → reads values
  - Alternative: use SheetJS (xlsx.js) loaded in the dialog HTML via CDN to parse client-side
  - Recommended: client-side SheetJS parse → send parsed 2D array JSON to handleStatementUpload()
    This avoids server-side binary parsing entirely

- Sheet.getImages() vs getCellImages():
  - getImages()     → OverGridImage[] — "Insert → Image → Insert image OVER cells"
  - getCellImages() → CellImage[]     — "Insert → Image → Place image IN cell"
  - This system uses getImages() ONLY — document clearly in Help text and initConfigTemplate()

- OAuth2 token caching:
  - PropertiesService.getScriptProperties().setProperty(TOKEN_CACHE_KEY, JSON.stringify({token, expiresAt}))
  - expiresAt = Date.now() + (expires_in * 1000) — TOKEN_EXPIRY_BUFFER
  - Always check cache before making OAuth call

- UrlFetchApp payload: max ~50MB — warn user if image file > 5MB

- getValues() returns 2D array — never flatten; always access as array[row][col]

- VLOOKUP formula must use absolute references for the Config range
  - Determine range at runtime from getConfig() → config.accountCodeRange
  - Example: =IFERROR(VLOOKUP(Q2,Config!$A$8:$B$30,2,0),"")

## ERROR HANDLING PATTERN (apply to ALL external calls)
```javascript
function callExternal(params) {
  try {
    const resp = UrlFetchApp.fetch(url, options);
    const code = resp.getResponseCode();
    if (code !== 200) {
      logMsg(LOG.X, 'ERROR', `HTTP ${code}`, resp.getContentText().substring(0, 200));
      return { error: `HTTP ${code}: ${resp.getContentText().substring(0, 100)}` };
    }
    const data = parseResponseData(resp.getContentText());
    return { data };
  } catch (e) {
    logMsg(LOG.X, 'EXCEPTION', e.message);

  }
}

## DEBUG GUIDE

**Failure 1: "No images found" even though images are visible in DD-MM sheet**
- Symptom: Phase A log shows "0 images found"
- Cause: Images inserted via "Insert → Image → Place image IN cell" (getCellImages() API)
- Diagnosis: In Script Editor, run: `Logger.log(SpreadsheetApp.getActiveSheet().getImages().length)`
  Also try: `Logger.log(SpreadsheetApp.getActiveSheet().getCellImages?.().length)` 
- Fix: Delete in-cell images, re-insert via "Insert → Image → Insert image OVER cells"
  Add detection to show helpful error: "Found 0 over-cell images. Found N in-cell images (not supported)."

**Failure 2: Currency mismatch — VND invoice not matching VND statement row**
- Symptom: VND invoice goes to /Unmatched/ despite matching row visible in sheet
- Diagnosis: Check Col H value in sheet — may have trailing space or wrong encoding
  Run Logger: `Logger.log(JSON.stringify(sheet.getRange(2, COL.CURRENCY_LOCAL).getValue()))`
  Check Gemini output currency field — may return "vnd" (lowercase) not "VND"
- Fix: Ensure currency comparison is `.toUpperCase().trim()` on both sides

**Failure 3: Gemini returns more/fewer items than images sent**
- Symptom: parseGeminiBatchResponse() count mismatch warning
- Cause: Gemini sometimes merges or splits invoice detections
- Diagnosis: Log raw Gemini response to verify array length vs input count
- Fix: Match by `imageIndex` field (not array position) — Gemini must return imageIndex per item
  If imageIndex missing: attempt positional fallback, log warning, do not throw
